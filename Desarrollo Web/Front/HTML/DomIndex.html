Preguntas

    1. ¿Qué es el DOM?
      • Respuesta Copilot: El DOM (acrónimo de Document Object Model) es una interfaz de programación de aplicaciones (API) para documentos HTML, XML y SVG.
      • Respuesta ChatGPT: En el contexto del Document Object Model (DOM), un nodo es un objeto que representa un elemento, atributo o texto en el árbol jerárquico que representa un documento HTML o XML.
    2. ¿Qué es un nodo?
      • Respuesta Copilot: Un nodo es un objeto que representa un elemento, atributo o texto en el árbol jerárquico que representa un documento HTML o XML.
      • Respuesta ChatGPT: En el contexto del Document Object Model (DOM), un nodo es un objeto que representa un elemento, atributo o texto en el árbol jerárquico que representa un documento HTML o XML.
    3. ¿Cómo se accede a un elemento del DOM?
      • Respuesta Copilot: Para acceder a un elemento del DOM, necesitas seleccionar el nodo que representa ese elemento utilizando JavaScript u otro lenguaje de programación que sea compatible con el DOM. Puedes hacer esto utilizando métodos como getElementById(), getElementsByClassName(), getElementsByTagName(), querySelector(), querySelectorAll().
      • Respuesta ChatGPT: Para acceder a un elemento del DOM, se puede utilizar JavaScript u otro lenguaje de programación que sea compatible con el DOM.
    4. ¿Cómo se modifica un elemento del DOM?
      • Respuesta Copilot: Se modifica un elemento del DOM con innerHTML, innerText, textContent, value, setAttribute, removeAttribute, appendChild, removeChild, replaceChild, insertBefore.
      • Respuesta ChatGPT: Para modificar un elemento del DOM, primero se debe seleccionar el nodo que representa ese elemento utilizando JavaScript u otro lenguaje de programación que sea compatible con el DOM.
    5. ¿Cuál es la diferencia entre getElementby* y querySelector*?
      • Respuesta Copilot: getElementById() devuelve un elemento con el id especificado, mientras que querySelector() devuelve el primer elemento que coincida con el selector CSS especificado.
      • Respuesta ChatGPT: La principal diferencia entre ellos es que getElementby* devuelve un solo elemento que coincide con el selector proporcionado, mientras que querySelector* devuelve una lista de elementos que coinciden con el selector.
    6. ¿Qué es un evento?
      • Respuesta Copilot: Un evento es cualquier cosa que suceda en el navegador. Por ejemplo, cuando el usuario hace clic en un elemento, mueve el mouse sobre un elemento, presiona una tecla, etc.
      • Respuesta ChatGPT: Un evento es una acción o suceso que ocurre en un documento web, como hacer clic en un botón, mover el cursor del ratón, presionar una tecla, cargar una página, etc.
    7. ¿Qué es un listener?
      • Respuesta Copilot: Un listener es un objeto que escucha los eventos de un objeto específico. Cuando se produce un evento, el listener se ejecuta.
      • Respuesta ChatGPT: Un listener, también conocido como "escuchador de eventos" o "manejador de eventos", es una función que se encarga de manejar los eventos en un documento web.
    8. ¿Qué es un callback?
      • Respuesta Copilot: Un callback es una función que se pasa a otra función como un argumento, que luego se invoca dentro de la función externa para completar alguna acción.
      • Respuesta ChatGPT: Un callback es una función que se pasa como argumento a otra función y que se invoca dentro de esa función para realizar alguna tarea o acción en particular.
    9. ¿Qué es un objeto de evento?
      • Respuesta Copilot: Un objeto de evento es un objeto que contiene información sobre un evento específico. Por ejemplo, cuando se produce un evento de clic, el objeto de evento contiene información sobre el elemento en el que se produjo el evento, la posición del cursor del ratón, etc.
      • Respuesta ChatGPT: Un objeto de evento es un objeto que contiene información sobre un evento que ha ocurrido en el navegador.
